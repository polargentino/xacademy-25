    Ejercicio Github con Bash
    Fecha de entrega: No hay fecha de entrega
    Puntos: 0

    Consigna:
    1. Crea un nuevo repositorio público en GitHub llamado loremipsum.
    2. Clona el repositorio en tu máquina local.
    3. Crea una nueva rama llamada generarlipsum.
    4. Agrega el archivo lipsum.sh provisto en esta tarea.
    5. Genera 5 archivos .txt usando lipsum.com ejecutando:
    bash ./lipsum.sh (asegúrate de tener curl instalado con sudo apt install curl).
    6. Verifica que se hayan creado los 5 archivos y que tengan contenido.
    7. Crea un commit con los archivos generados y lipsum.sh, luego haz push a GitHub.
    8. Crea un nuevo archivo contar.sh que lea cada .txt y muestre cuántas líneas tiene cada uno.
    9. Ejemplo de ejecución:
    bash ./contar.sh
    loremipsum-1.txt tiene 4 líneas.
    loremipsum-2.txt tiene 7 líneas.
    loremipsum-3.txt tiene 15 líneas.
    loremipsum-4.txt tiene 7 líneas.
    loremipsum-5.txt tiene 16 líneas.
     10. Haz un nuevo commit en la rama generarlipsum sumando contar.sh.
    11. Realiza un merge de generarlipsum a la rama principal mediante una Pull Request.

    Comandos útiles: echo, wc, man {comando}, curl cheat.sh/{comando}.

    Archivo lipsum.sh:

    #!/bin/bash
for i in {1..5}
do
    lineas=$(shuf -i 1-20 -n 1)
    lorem=$(curl -s https://www.lipsum.com/feed/xml?amount=$lineas | sed -n '//,/<\/lipsum>/p' | sed -e 's///g' -e 's/<\/lipsum>//g')
    echo "$lorem" >> loremipsum-$i.txt
done

Ejercicio Git y GitHub - Información del Alumno

  1. Hacer un fork del repositorio SantexGroup/SantexAcademy en tu cuenta personal de GitHub.
  2. Clonar el repositorio forkeado en tu ambiente local usando SSH: 
    git clone git@github.com:TU_USUARIO/SantexAcademy.git
  3. Crea una nueva rama con el nombre informacion-alumno usando el comando: 
    git branch informacion-alumno
  4. Cambia a la rama que acabas de crear usando el comando: 
    git checkout informacion-alumno
  5. Abre el archivo README.md en tu editor de texto preferido y agrega tu nombre, apellido, nacionalidad y email a la sección de información del proyecto.
  6. Agrega los cambios que hiciste al archivo README.md a la zona de preparación usando el comando: 
    git add README.md
  7. Crea un nuevo commit con un mensaje descriptivo usando el comando: 
    git commit -m "Agrega información del alumno al README"
  8. Sube tus cambios a GitHub usando el comando: 
    git push origin informacion-alumno
  9. Crea una Pull Request en GitHub para fusionar tus cambios con la rama principal del repositorio.

Ejercicio Git y GitHub - Guía Paso a Paso

1. Hacer un fork del repositorio

Ve al repositorio SantexGroup/SantexAcademy en GitHub y haz clic en el botón Fork (arriba a la derecha). Selecciona tu cuenta personal como destino.

2. Clonar el repositorio forkeado

Ve a tu repositorio forkeado en GitHub, copia la URL SSH y ejecuta en tu terminal:

git clone git@github.com:TU_USUARIO/SantexAcademy.git
cd SantexAcademy

3. Crear una nueva rama

Crea la rama informacion-alumno:

git branch informacion-alumno

4. Cambiar a la nueva rama

Cambia a la rama recién creada:

git checkout informacion-alumno

5. Editar el archivo README.md

Abre el archivo README.md y agrega tu información (nombre, apellido, nacionalidad y email). Ejemplo:

## Información del Alumno
- Nombre: Tu Nombre
- Apellido: Tu Apellido
- Nacionalidad: Argentina
- Email: tu.email@example.com

6. Agregar los cambios al área de preparación

Prepara los cambios para el commit:

git add README.md

7. Crear un commit

Guarda los cambios con un mensaje descriptivo:

git commit -m "Agrega información del alumno al README"

8. Subir los cambios a GitHub

Sube la rama a tu repositorio remoto:

git push origin informacion-alumno

9. Crear una Pull Request

Ve a tu repositorio forkeado en GitHub y haz clic en Compare & pull request. Completa el título y la descripción, y crea la Pull Request.
